CSS: Setup

The project uses Sass to provide the CSS with some structure.

  • You need to install Ruby and Sass.
  • Watching and compiling is done with one single command, preferably outside the vagrant box: 
    sass --watch assets/static/sass/style.scss:assets/static/css/style.css.
  • Don't edit the generated style.css in /assets/static/css
CSS: Structure

The partials folder inside /assets/static/sass contains all the small Sass files that are imported into the style.scss folder.

Important partials include:

  • Variables: Contains all variables (or constants) referenced in all other Sass files. A variable is always prefixed with a $ symbol. e.g. $header-height defines the height of the fixed height header, which is then reused for positioning the main element, map, fixed alerts, etc.
  • Globals: vendor prefixing mixins and global classes belong to the globals partial. These affect the entire app; handle with care if making changes.
  • Typography: Contains styles for headings, links, lists, etc. These affect the entire app; handle with care if making changes.
  • Forms: Contains styles for inputs, labels, selects, errors. All overrides for individual forms are also done here.
  • Z-index: Contains all the z-index values for all selectors, for easy reference. Add new z-index values here.
  • Map: Contains all leaflet map styles and overrides, including pop-ups and filters.
  • Media-queries: Contains all the styles that make the app responsive.
  • Lib folder Contains all third party plugin sass partials

Most changes to pages or new pages can be done in the _main.scss or _media-queries partials, with helpful comments to indicate the page name

Don't carelessly edit the variables, globals or typography files, since those style changes will cascade throughout the site.

DO: create a new partial, if you need a new UI module that deserves its own partial file, e.g. some groovy animations. The partials file should always be named _<file-name>.scss and import it into the style.scss file like this. 

@import 'partials/groovy.animation';

As you can see above, in the import statement, neither the underscore nor the .scss extension of the file name is specified.

Import documentation here.

CSS: Conventions
  • Do use 2 spaces to indent.
  • Don't use ids for styling; they are used for JS
  • Do use variables, nesting, helper classes, mixins, extends and placeholders in a manner that benefits the app. Some documentation if you're new to Sass.
  • Do use Sass comments (similar to JS single line comments //) for comments that only other developers need to see. These do not get compiled into the final CSS stylesheet.
  • Do use helpful comments for new sections inside a Sass partial, which are useful for documentation of the section in question. See below: 
    /*-------------------------------------*/
/* FORM: BASE STYLES */
  • Do spend a few minutes browsing through the globals, variables and typography partials. They contain plenty of reusable code.
  • Don't try to extend a Sass placeholder (e.g. %grid-parent) inside a media query, because you will get an error and the file won't compile. If a mixin is available, use that instead, inside the media query block.

Grid: Media Queries

By and large, the site uses min-width media queries, on the assumption that the initial layout is mobile-first and stacked, and columns are added incrementally, as the screen size gets larger.

  • Media queries are added after the selector in question.
  • Media queries are available as Sass mixins and can be nested, which is an advantage.
  • Do add min-width media queries in the order of smallest to largest.
  • The 4 min-width media queries available as mixins are:
    • media-sm: 600px
    • media-med: 750px
    • media-lg: 990px
    • media-lg-x: 1200px (used sparingly)
  • The 1 max-width media query is 749px. This is used sparingly.

Grid: Classes

There's no real grid, in case you were wondering. But there's enough stuff to build flexibly.

You can add more classes as required in the grid partial; the current grid simply accommodates 1/2, 1/3rd, 2/3rds and 1/4th widths.

Building Blocks

  • Firstly, add a section with a container class or mixin or placeholder. (This is optional if a parent container class already exists in the DOM).
  • Next, add a parent for your grid: with a class of grid-parent (this internally has a clearfix, so you don't need to worry about clearing floats).
  • Add some children with a class of grid-child, to get some sweet grid gutters.

Mixins instead of grid classes

The demos above will work for sections that don't need to be stacked or have their grids changed on different media breakpoints.

Do create responsive grids using the recommendations below.

Since the site is built mobile first, it would be preferable not to add classes, as indicated in the demos above, since they would have to be reset for mobile, to become full-width sections (in most cases, but not all).

Instead, consider using the handy grid mixins available on media query breakpoints. The mixins have the same names as the grid classes, i.e. grid-parent, grid-child, col-25-flt, col-50-flt, col-33-flt, col-66-flt and grid placeholders.

Let's look at a simple demo below, using classes from the existing site. On a medium media query breakpoint, the parent class partners-list-block includes the mixin grid-parent, and the children partners-list include the mixin col-50-flt.

Thus you can float columns of any widths (that add up to 100%) on any media query, to get a truly flexible grid.

Icons

Documentation

The site uses the Font Awesome icon collection. All icons are vector and used and styled like a font.

Useful variables and helpers for icons include:

$font-icon / .font-icon / %font-icon; //extends Font Awesome
.icon-left or %icon-left // adding padding to the right for icons on the left
.icon-right or %icon-right // adding padding to the left for icons on the right

The documention for icons is here and the cheatsheet is here.

To use icons in the CSS, you will need to catch the unicode value in the cheatsheet, escape it and add it as generated content using content :before.

<style>
.icon-boy {
  &:before {
    content: "\f183";
    font-family: $font-icon;
  }
}
</style>

To use icons in the markup, add a class of fa and fa-<icon-name> to an element.
